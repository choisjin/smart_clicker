# Unit Tracking Mode — Plan

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 게임 화면에서 특정 유닛을 수동으로 찾아 우클릭하는 반복 작업이 비효율적 |
| **Solution** | 사용자가 유닛을 한 번 박싱하면, 실시간 프레임에서 특징 매칭으로 자동 추적+우클릭 |
| **Function UX Effect** | 추적 셋팅 → 드래그 박싱 → 추적 시작 → 자동 우클릭 (3단계 UX) |
| **Core Value** | 반복 클릭 작업 자동화, 사람처럼 자연스러운 HID 입력 유지 |

---

## 1. 기능 개요

### 1.1 추적 모드 셋팅
- GUI에서 "추적 셋팅" 버튼 클릭
- 현재 스트리밍 프레임을 **정지 화면**으로 별도 창에 표시
- 사용자가 마우스 드래그로 추적할 유닛을 **타이트하게 바운딩 박스** 선택
- 선택 영역에서 유닛 특징 추출 (Feature Extraction)

### 1.2 특징 추출
- **색상 히스토그램** (HSV 공간, 배경과 구분)
- **템플릿 이미지** (선택 영역 자체를 템플릿으로 저장)
- **형태 특징** (윤곽선 면적, 종횡비)
- 추출된 특징을 "추적 프로파일"로 저장

### 1.3 실시간 추적
- 스트리밍 프레임 수신 시마다 특징 매칭 수행
- 매칭 방법:
  - **템플릿 매칭** (cv2.matchTemplate, 멀티스케일)
  - **색상 히스토그램 역투영** (cv2.calcBackProject)
  - 두 방법의 가중 평균으로 일치도 계산
- 일치도 임계값 이상인 후보들 검출
- **NMS** (Non-Maximum Suppression)로 중복 제거

### 1.4 자동 우클릭
- 검출된 일치 유닛 중 **화면 중앙에서 가장 가까운** 유닛 선택
- Arduino HID로 해당 좌표에 우클릭 전송
- 우클릭 후 일정 쿨다운 대기 (연속 클릭 방지)
- 추적 계속 → 다음 일치 유닛 → 반복

## 2. 기술 설계

### 2.1 아키텍처

```
GUI (Control PC)                    Agent (Target PC)
─────────────────                   ──────────────────
[추적 셋팅 버튼]
     ↓
[스크린샷 정지 창]
     ↓ (드래그 박싱)
[특징 추출] ← OpenCV
     ↓
[추적 프로파일 저장]
     ↓
[추적 시작]
     ↓
[프레임 수신] ←─── 스트리밍 ───── [캡처]
     ↓
[특징 매칭] ← OpenCV
     ↓
[가장 가까운 유닛 좌표 계산]
     ↓
[우클릭 명령] ───── 명령 WS ────→ [HID 우클릭]
```

### 2.2 처리 위치
- **특징 추출/매칭**: Control PC (GUI) — Agent에 부하 안 줌
- **캡처/스트리밍**: Agent (Target PC) — 기존 그대로
- **우클릭 실행**: Agent → Arduino HID

### 2.3 주요 클래스

| 클래스 | 파일 | 역할 |
|--------|------|------|
| `TrackingProfile` | `tracking.py` | 추출된 특징 저장 (템플릿, 히스토그램, 임계값) |
| `UnitTracker` | `tracking.py` | 프레임에서 특징 매칭 → 후보 좌표 반환 |
| `TrackingSetupDialog` | `gui/tracking_dialog.py` | 스크린샷 정지 + 드래그 박싱 UI |
| `TrackingController` | `gui/main_window.py` | 추적 시작/중지, 프레임마다 매칭+우클릭 오케스트레이션 |

### 2.4 매칭 파라미터

| 파라미터 | 기본값 | 설명 |
|----------|--------|------|
| `match_threshold` | 0.7 | 일치도 임계값 (0.0~1.0) |
| `template_weight` | 0.6 | 템플릿 매칭 가중치 |
| `histogram_weight` | 0.4 | 히스토그램 매칭 가중치 |
| `cooldown_sec` | 1.0 | 우클릭 후 쿨다운 (초) |
| `scale_range` | (0.8, 1.2) | 멀티스케일 범위 |
| `nms_threshold` | 0.3 | NMS 중복 제거 임계값 |

## 3. 구현 순서

### Phase 1: 추적 셋팅 UI
1. `TrackingSetupDialog` — 스크린샷 정지, 드래그 박싱, 영역 반환
2. AgentPanel에 "추적 셋팅" 버튼 추가

### Phase 2: 특징 추출
3. `TrackingProfile` — 선택 영역에서 템플릿+히스토그램 추출
4. 추출 결과 미리보기 (매칭 테스트)

### Phase 3: 실시간 매칭
5. `UnitTracker` — 프레임 입력 → 후보 좌표 반환
6. GUI에서 프레임 수신 시마다 매칭 실행 (별도 스레드)

### Phase 4: 자동 우클릭
7. 매칭 결과 중 화면 중앙 최근접 계산
8. `send_click_to_window` 우클릭 전송
9. 쿨다운 타이머

### Phase 5: 추적 시각화
10. GUI 스트리밍 화면에 추적 박스 오버레이 표시
11. 일치도 수치 표시

---

## 4. 제약 사항

- 게임 해상도/DPI 변경 시 추적 프로파일 재설정 필요
- 유닛 방향/애니메이션 변화가 크면 매칭 정확도 하락 가능 → 멀티스케일+히스토그램 보완
- 매칭 연산이 무거우면 프레임 드롭 → 매칭 주기를 FPS와 분리 (예: 10Hz)
- Arduino HID 우클릭은 xgincode 안전 (하드웨어 입력)
